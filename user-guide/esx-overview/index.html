Docker volumes on ESX¶

Docker volumes on vSphere are powered by VMDKs. VMDKs can reside on datastores created on top of varying backends (NFS, SAN, VSAN, vVol). Docker-volume-vsphere plugin supports high availability for Docker volumes and allows for any VM requesting for the volume to gain access to it. The ability to attach the VMDK(Docker volume) to any VM when paired with a cluster manager such as Swarm allows a persistent container to be highly available.

The VMDKs on ESX are stored on the datastore in a folder that is independent of a VM.
